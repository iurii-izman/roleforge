# Contributing

## Workflow

1. Start with a small, well-scoped change.
2. Inspect the current repository state before editing.
3. Document assumptions in the PR or commit message.
4. Update docs when the product or architecture direction changes.
5. Keep secrets out of the repository.

## Branch Naming

- `feat/<short-name>`
- `fix/<short-name>`
- `docs/<short-name>`
- `chore/<short-name>`

## Commit Style

Use Conventional Commits when practical:

- `feat:`
- `fix:`
- `docs:`
- `chore:`

## Pull Requests

- Keep PRs focused and reviewable.
- Mention what was AI-assisted if that context matters for review.
- Include validation steps, even if they are manual.
- Call out skipped tests or follow-up work explicitly.
